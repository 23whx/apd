ACGN Personality Database — 项目文档（Draft v1）

一个可由用户提交 ACGN 作品并对角色人格进行投票的多语言数据库网站
UI：EVA 配色 · 技术栈：React + Tailwind + Supabase + Firecrawl + DeepSeek/通义千问

1. 项目简介

ACGN Personality Database 是一个类似 Personality-Database 的社区型人格投票平台。
用户可以提交 ACGN 作品（动漫、游戏、轻小说、漫画），系统自动通过爬虫抓取基础信息，并允许用户对作品内角色进行多种人格系统的投票。

目标：构建一个精简、高速、多语言的 ACGN 角色人格数据库。

2. 项目核心特性
2.1 多语言支持（中/英/日）

每个作品与角色条目需存储：

中文名

英文名

日文名

用户可切换网站语言（i18n：推荐 react-i18next）。

2.2 用户提交 ACGN 作品（含 LLM 消歧）
提交流程

用户输入一个作品名称（任意语言皆可）。

LLM（DeepSeek 或 通义千问）进行：

同名/别名检查：判断是否为数据库中已有作品的别名

误导性提交识别（例如恶搞、重复、伪造名称）。

判定是否新作品 → 新则进入下一步。

自动调用 Firecrawl 爬取：

萌娘百科

维基百科

百度百科

Firecrawl 返回内容后，仅保留：

作品多语言名称（中英日）

角色名称（中英日）

角色外链图片 URL（不存文件）

原文档链接（外链）

（可选）角色在百科中的链接

❗不存储图片文件、不保存长篇介绍文本（因 Supabase 容量有限）。

2.3 人格学投票系统

为每个角色展示投票维度：

MBTI（16 类型）

九型人格（含侧翼）

九型副型（SP/SO/SX）

易学人格学（由你自行定义类型表）

数据结构为简单的计数统计：

每次投票存入一个 record（user_id + character_id + 投票类型）

页面展示合计结果 / 比例图表

2.4 用户系统（简约）

注册/登录使用 Supabase Auth

用户头像：

不允许上传本地图片

网站提供固定头像库（例如 12~24 个 EVA 风格头像）

用户只能选择预设头像，提高隐私与性能

4. LLM 消歧逻辑（核心技术）

此项目中 LLM 的主要价值不在生成文本，而在 判断用户提交是否是同一部作品的不同名称。

LLM 判断点

输入： 用户提交名称
输出：

same_work_id（若为别名则返回对应 ID）

is_misleading（如用户提供恶搞名称 → true）

canonical_name_zh/en/jp（规范化名称）

判断策略

将用户输入与数据库当前条目进行全语言比较

调用百科爬虫

使用 LLM 对比百科结果与数据库现有信息

若高度相似 → 当作别名合并

若恶搞（如“进击的肥龙”） → 拒绝

若确实是新作品 → 添加新条目

5. Firecrawl 爬虫策略

Firecrawl 来源：

萌娘百科

百度百科

中文维基百科

英文维基百科（必要时）

提取内容限制：

仅保留名称（中英日）

角色列表（名称 + 图片外链 + 百科链接）

不保存故事简介（浪费空间）
